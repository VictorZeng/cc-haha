# Claude Code Haha v0.4.10

这是 `v0.4.9` 之后的稳定性与恢复版本，重点修复会话重启、后台 Agent、权限拒绝、H5 断连、对话搜索和 Windows 覆盖升级中的边界问题，并补齐启动诊断、代理波动恢复、模型角色配置和无界面 H5 部署能力。

相比 `v0.4.9`，本版本更可靠地保存“已经完成、已经停止、已经拒绝”的真实状态。应用冷启动、网络波动或手机离开 H5 后，不应再把终态回退为运行中，也不应把未执行的写入显示为文件变更。Windows 安装器会更准确地区分真正占用安装目录的进程，并保护升级前的用户数据。

## Highlights

- **会话恢复与撤销更可信**：撤销当前轮次改用真实 checkpoint；重启前会先落盘上下文，恢复历史会话时不再误触发预热，长对话重新挂载后保持阅读位置。
- **后台任务终态持久化**：后台 Agent 的完成、失败和停止状态会写入会话历史；完全退出再启动后，已完成卡片不会回退成“进行中”，旧版通知格式仍可继续读取。
- **H5 断连清理补齐启动竞态**：手机在 CLI 启动阶段离开、权限请求随后才出现时，服务端仍会建立清理观察；无人连接的权限等待会在上限后停止，重连和正常长任务不会被误杀。
- **Windows 覆盖升级加固**：安装器更准确地检查相关进程、处理当前用户安装和旧数据恢复，减少 CLR HRESULT、误报程序仍在运行或必须卸载重装的问题。
- **对话搜索与滚动稳定**：虚拟化长对话支持可靠搜索、方向导航和当前匹配定位；快速输入、内容刷新与组件重新挂载不会让旧结果覆盖新结果或把滚动位置拉回底部。
- **代理、启动与诊断更可恢复**：系统代理连接重置不会再触发主进程 JavaScript 异常；渲染器入口失败时会显示启动错误；SDK 诊断消息设有保留上限，避免长时间运行后持续占用内存。

## Fixes

- 修复异常结束的轮次必须重启后才能撤销、或撤销到错误文件状态的问题；现在使用服务端权威 checkpoint，只回退目标轮次 (#1030)。
- 修复上下文在运行时重启前未完整落盘、恢复会话时额外预热可能污染上下文的问题 (#1033, #1054)。
- 修复长对话重新挂载后滚动位置丢失，以及搜索结果更新时方向键、当前匹配和高亮不同步的问题 (#1045, #1057)。
- 修复后台 Agent 完成通知与前台 Skill 并发时可能影响前台输出，以及冷启动后终态回退为“进行中”的问题 (#1047)。
- 修复中断的历史任务长期保持运行状态的问题；未完成轮次现在收敛为“已停止”，真正完成的轮次仍保持“已完成” (#1034)。
- 修复普通工具被拒绝后当前轮次仍继续执行，以及拒绝的 Write/Edit 虽未落盘却出现在 changed-files 卡片和工作区变更中的问题 (#1051)。
- 修复 H5 已离开但权限请求稍后才出现时，清理观察未能生效的问题；同时保留重连取消清理和正常后台执行语义 (#1019)。
- 修复系统代理节点切换、短暂断网或连接重置时弹出主进程异常、恢复网络后仍需重启的问题 (#1050, #1056)。
- 修复 Windows 当前用户覆盖安装可能出现 CLR HRESULT、误判残留进程、错误阻止安装或损伤旧数据恢复的问题 (#1028, #1029, #1036)。
- 修复渲染入口加载失败时应用只剩后台进程、没有可见错误窗口的问题 (#1032)。
- 修复 Sidecar 长时间保留过多 SDK 诊断事件导致内存持续增长的问题，同时保留实时回复和关键错误信息 (#1049)。
- 修复应用缩放和窗口变化后，原生浏览器预览与 React 容器边界错位的问题 (#1042)。
- 修复关闭最后一个预览标签后标签仍残留、需要额外操作才能退出预览的问题 (#1038)。
- 修复附件参与上下文分析时，不同消息形态可能导致状态计算不一致的问题 (#1022)。

## Features and Improvements

- 模型选择器会合并运行时配置中的主模型和 Haiku、Sonnet、Opus 等角色模型，并在重启后保持当前模型 (#1024)。
- 对话内查找新增面向虚拟化消息列表的定位桥接，可在大量历史消息中稳定查找并跳转。
- H5 文档补充无界面服务器构建和部署路径，发布流程会把 Web UI 产物包含进 Sidecar 构建 (#1027)。
- Proxy Trace 写盘与响应完成解耦，慢磁盘或大 Trace 不再阻塞用户看到最终回复 (#1014)。
- WebSocket 与会话服务增加后台任务终态的顺序写入、去重和旧格式兼容，多个观察端不会重复落盘同一通知。
- Windows 发布 smoke 增加安装进程检查、默认模式重装和旧目录数据恢复覆盖。

## 安装

在本页面底部 Assets 中按系统和 CPU 架构下载：

| 平台 | 文件 |
|------|------|
| macOS Apple Silicon（M 系列） | `Claude-Code-Haha-0.4.10-mac-arm64.dmg` |
| macOS Intel | `Claude-Code-Haha-0.4.10-mac-x64.dmg` |
| Windows x64 | `Claude-Code-Haha-0.4.10-win-x64.exe` |
| Windows ARM64 | `Claude-Code-Haha-0.4.10-win-arm64.exe` |
| Linux x64 | `Claude-Code-Haha-0.4.10-linux-x86_64.AppImage` 或 `Claude-Code-Haha-0.4.10-linux-amd64.deb` |
| Linux ARM64 | `Claude-Code-Haha-0.4.10-linux-arm64.AppImage` 或 `Claude-Code-Haha-0.4.10-linux-arm64.deb` |

### macOS

双击 DMG，把应用拖入“应用程序”。正式签名和公证产物应直接通过正常 Gatekeeper 流程。

### Windows

双击 `.exe` 安装。建议直接以当前用户运行，不要使用“以管理员身份运行”。覆盖升级前先完全退出应用；如果安装器报告相关进程仍在运行，请根据提示关闭对应进程后重试。未签名构建出现 SmartScreen 时可选择“更多信息”→“仍要运行”。

### Linux

AppImage：

```bash
chmod +x Claude-Code-Haha-0.4.10-linux-x86_64.AppImage
./Claude-Code-Haha-0.4.10-linux-x86_64.AppImage
```

deb：

```bash
sudo apt install ./Claude-Code-Haha-0.4.10-linux-amd64.deb
```

ARM64 机器请换用对应的 `linux-arm64` 文件。
